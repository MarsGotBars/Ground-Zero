Poisonous animals; there is plenty of them to go around; but what if we wanted to _select_ a specific one!

## What came before

Before, `select` elements on the web appeared in 2 formats:

- [Ugly and (sometimes) minimally styled](#base) (which is completely omitted on mac...).
- Packages / custom implementations using plenty of JS (ew).

As it stands people often opt for the latter, non-native solutions like building their own `select` elements from scratch and filling in all (baseline) necessary functionality and accessibillity, a long and arduous task.

## What the present offers

But presently; `select` elements are steadily gaining (support _(already sitting at around 70%!)_ and-) the possibility for customization; and in a variety of ways!

Allowing us to finally build the select elements of our dreams, completely stylable, inside & outside!

- Allowing for changes to the default picker-icon (the little arrow pointing down)
 - You can completely swap this with a custom svg using a `background-image`.
- Styling the element that contains the displayed options when a select is opened!
- Styling the currently selected option.
- It even contains the possibility for a checkmark; of which we can, for example, adjust the position with the `position` property that you are already used to!
- It even includes some `anchor()` positioning; allowing you to anchor the options however you'd like to the `select`
And a bunch more!
